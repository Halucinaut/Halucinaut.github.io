## 问题意识

工具智能体在长期运行中经常遇到相似任务，但常见记忆方案主要依赖上下文摘要、向量检索和最近历史。这些机制能找回相关内容，却难以判断历史经验是否真正帮助过任务，也缺少把失败经验、过时经验和低价值记录降权的闭环。

UMI-TIDE 的核心观察是：**长期记忆的核心风险不是"记得不够多"，而是"无法知道什么值得再次使用"**。

## 核心洞察

T0 阶段实现闭环机制：

```
任务执行 → 记录候选经验 → 任务后打分 → 写入记忆库 → 多目标检索 → 低价值记录废弃
```

- **MemoryRecord**：结构化经验单元，保存任务类型、动作、结果、成本、收益分数、置信度、复用次数、冲突分数
- **UtilityScorer**：任务结束后用启发式权重更新收益分数和状态
- **Retriever**：综合语义相似度、收益分数、置信度、任务类型匹配、时间衰减和冲突惩罚排序
- **ForgettingPolicy**：将低收益、低置信度或高冲突记录标记为 deprecated

## 与 Related Work 的差异

| 方案 | 关注点 | UMI-TIDE 差异 |
|---|---|---|
| RAG / 向量记忆 | 语义相关性 | 把语义相似度作为排序信号之一，同时显式加入收益分数、任务匹配、时间衰减和冲突惩罚 |
| 摘要记忆 | 文本压缩 | 保存结构化动作片段和结果标签，保留工具参数、失败动作、成本和复用记录 |
| 反思日志 | 人工整理 | 验证外部经验记录能否通过结果信号完成写入、检索和遗忘闭环 |
| 经验回放 | 模型训练 | 只更新外部记忆记录的分数、状态和复用次数，不训练基础模型 |

## 当前状态

T0 已实现：经验数据结构、本地 JSON 存储、收益打分、词袋检索、低价值记录废弃、确定性工具智能体、六组基准对照、单元测试。

六组对照模式：`no_memory`、`semantic`、`semantic_recency`、`semantic_summary`、`value`、`value_forgetting`
